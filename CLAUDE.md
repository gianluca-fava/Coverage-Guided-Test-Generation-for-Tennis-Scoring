# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Essential Commands

All commands assume the virtualenv is active (`source venv/bin/activate`) and are run from the project root.

```bash
# Run sanity test suite
python -m pytest tests/ -v

# Run tests with branch coverage report
python -m pytest tests/ --cov=src/tennis_engine --cov-branch --cov-report=term-missing

# Smoke test (3 seeds, ~2 min)
python -m experiments.run_experiments --smoke --budget 200

# Full experiment (30 seeds, ~40 min)
python -m experiments.run_experiments --n 30 --budget 200

# Run only specific generators (e.g. after fixing a crash mid-run)
python -m experiments.run_experiments --n 30 --budget 200 --generators search ablation

# Statistical analysis + figures (reads results/results.csv)
python -m experiments.analyze
```

## Architecture

### Internal contract (`CONTRACT.md`)
All modules communicate through a strict interface. `TennisMatch.score()` returns a dict with exactly these keys: `points`, `games`, `sets`, `in_tiebreak`, `is_over`, `winner`. The `measure_branch_coverage()` function returns `branch_coverage`, `covered_branches`, `total_branches`. Results go to `results/results.csv` with columns `generator,ruleset,seed,branch_coverage,mutation_score,time_sec`.

### SUT (`src/tennis_engine.py`)
Three rulesets (`wimbledon`, `usopen`, `ausopen`) differ only in the decisive-set tie-break target: Wimbledon=10pts, US Open=7pts, AusOpen=10pts. The engine uses explicit `if/elif` branches throughout (intentional, for branch diversity). `play_point()` raises `ValueError` for invalid winner and `RuntimeError` if called after match ends.

### Coverage tool (`src/coverage_tool.py`)
Uses `coverage.py` JSON report API (not the internal `_analyze` API, which changed in v7). Targets only `src/tennis_engine.py`. Each call starts a fresh `Coverage` instance. Returns branch arcs as `[from_line, to_line]` pairs from the JSON.

### Generators
- `random_generator.py`: generates `budget` independent random sequences; fully deterministic given `seed`.
- `search_generator.py`: evolutionary algorithm; fitness = new branch arcs covered beyond running archive. Archive accumulates all evaluated sequences and is what gets passed to `measure_branch_coverage` at experiment time. The `_mutate` operator checks `room = max_len - len(seq) >= 1` before trying to extend (important — crash if missing).
- `search_generator_ablation.py`: identical structure but fitness always returns 0 (no guidance); isolates contribution of the fitness function.

### Mutation score (`experiments/run_experiments.py`)
Uses **mutmut v3** (not v2 — CLI is completely different). Key constraints:
- Config in `setup.cfg` under `[mutmut]` with keys `source_paths` and `pytest_add_cli_args_test_selection`.
- Generated tests write to `tests/test_generated_temp.py` (fixed path, cleaned up after each run).
- Tests must import as `from tennis_engine import TennisMatch` (not `from src.tennis_engine`) because mutmut v3's trampoline mechanism fails if module name starts with `src.`. Path is set with `sys.path.insert(0, os.path.join(os.getcwd(), 'src'))`.
- Only `.mutmut-cache` is deleted between runs; `mutants/` directory is kept (same source = same mutants, saves regeneration time).
- Parse results with `mutmut results --all true` and count `": killed"` vs `": survived"` strings.

### Experiment flow
`run_experiments.py` → for each (generator, ruleset, seed): generate sequences → `measure_branch_coverage` → `compute_mutation_score` → append row to CSV. The `--generators` flag filters which generators run (useful for resuming after a crash).

`analyze.py` → reads CSV → boxplots per ruleset + combined mutation score boxplot → Mann-Whitney U (scipy) + A12 effect size (implemented inline) → summary CSV and TXT.

## Git rules
- No "Co-Authored-By" or AI-related trailers in commit messages.
- Always confirm with user before pushing.
- `mutants/` is gitignored (generated by mutmut at runtime).
- `results/` is NOT gitignored (results are part of the deliverable).
